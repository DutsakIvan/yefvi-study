8-12%1. Алгоритми та обчислювальна складність

Алгоритм — це скінченна послідовність однозначних інструкцій для розв'язання задачі. Ми оцінюємо алгоритми за часом (time complexity) і пам'яттю (space complexity).

Асимптотична складність — O-нотація B

Асимптотична складність показує, як зростає час виконання алгоритму зі збільшенням розміру вхідних даних n. Нас цікавить найгірший випадок і порядок зростання.

Чим крутіша крива — тим гірше. O(1) — ідеал, O(n!) — катастрофа.

Візуалізатор сортування B

Кожен стовпець — елемент масиву, число над ним — його значення. Жовтий = ці два зараз порівнюються. Червоний = ці два міняються місцями. Зелений = вже на правильному місці. Читай рядок статусу — там пояснюється кожен крок. Змінюй швидкість повзунком під час роботи — ефект миттєвий.

Стратегії розроблення алгоритмів B

«Розділяй та володарюй» (Divide & Conquer) — розбиття задачі на підзадачі, рекурсивне розв'язання, комбінування. Приклади: Merge Sort, Quick Sort, двійковий пошук.

Динамічне програмування — розв'язання через комбінування розв'язків підзадач із запам'ятовуванням (мемоізація). Приклади: числа Фібоначчі, задача про рюкзак, найдовша спільна підпослідовність.

Жадібні алгоритми — на кожному кроці обирається локально оптимальний варіант. Приклади: алгоритм Дейкстри, алгоритм Крускала (MST), кодування Гаффмана.

Backtracking (пошук із поверненням) — рекурсивний перебір із відсіканням неперспективних гілок. Приклад: задача про N ферзів.

Класи складності P, NP, NP-повні B

Клас P — задачі, які можна розв'язати за поліноміальний час O(n^k). Наприклад: сортування, пошук найкоротшого шляху.

Клас NP — задачі, розв'язок яких можна перевірити за поліноміальний час. Наприклад: задача комівояжера — перевірити маршрут легко, знайти оптимальний — складно.

NP-повні задачі — найскладніші в NP. Якщо для однієї знайдеться поліноміальний алгоритм, то P = NP. Приклади: SAT, задача про рюкзак, розфарбування графа.

Алгоритми на графах B

BFS (пошук вшир) — використовує чергу. Знаходить найкоротший шлях у незваженому графі. O(V+E).

DFS (пошук вглиб) — використовує стек або рекурсію. O(V+E).

Алгоритм Дейкстри — найкоротші шляхи від однієї вершини у зваженому графі без від'ємних ребер. O((V+E) log V).

Алгоритми Крускала / Пріма — побудова мінімального кістякового дерева (MST).

8-10%2. Архітектура комп'ютера

Як комп'ютер представляє числа в пам'яті, виконує команди, організовує пам'ять і введення/виведення.

Конвертер систем числення B

Клацай біти — дивись, як змінюється десяткове, двійкове й шістнадцяткове значення. Доповнювальний код (Two's Complement) — стандарт для представлення цілих чисел у сучасних CPU. Старший біт = 1 → від'ємне число.

Доповнювальний код (8 біт): діапазон [−128, 127]. Поточне:

Представлення чисел у пам'яті B

Прямий код — старший біт = знак (0=+, 1=−), решта = модуль. Недолік: два нулі (+0 і −0).

Обернений код (Ones' Complement) — від'ємне число = інверсія бітів додатного. Також два нулі.

Доповнювальний код (Two's Complement) — від'ємне = інверсія + 1. Стандарт у всіх сучасних CPU. Один нуль. Для n біт: діапазон [−2ⁿ⁻¹, 2ⁿ⁻¹−1].

IEEE 754 (плаваюча кома) — float (32 біти: 1 знак + 8 експонента + 23 мантиса), double (64 біти: 1+11+52). Проблеми: 0.1+0.2 ≠ 0.3 через двійкове представлення.

Архітектура фон Неймана A

Принципи: двійкове кодування, програмне керування, однорідність пам'яті (команди + дані разом), адресність, послідовне виконання.

«Вузьке місце» (bottleneck) — обмежена пропускна здатність шини між CPU і пам'яттю. Процесор простоює, чекаючи дані.

Гарвардська архітектура — роздільна пам'ять для команд і даних (швидше, складніше). Використовується в мікроконтролерах і кешах L1.

Цикл CPU: Fetch (вибірка) → Decode (декодування) → Execute (виконання) → Writeback (запис).

Ключове для ЄФВІ: знати різницю між фон Нейманом і Гарвардом, принципи представлення цілих і дійсних чисел.

Організація пам'яті A

Ієрархія пам'яті (зверху вниз — швидше, менше, дорожче): Регістри → Кеш L1 → Кеш L2 → Кеш L3 → ОЗП (RAM) → SSD/HDD.

Кеш-пам'ять — принцип локальності: часова (щойно використане — знову знадобиться) і просторова (сусідні дані теж знадобляться).

Віртуальна пам'ять — програми «бачать» більше пам'яті, ніж є фізично. Реалізується через підкачку сторінок (paging) на диск.

2. Яка архітектура використовує спільну пам'ять для команд і даних?

Реляційна модель B

Запропонована Едгаром Коддом (1970). Дані — у вигляді таблиць (відношень).

Кортеж (Tuple) = рядок таблиці. Атрибут (Attribute) = стовпець. Схема = структура (імена + типи атрибутів).

Нормалізація B

1NF — атомарність: кожне поле містить одне неподільне значення.

2NF — 1NF + всі неключові атрибути залежать від повного первинного ключа (не від частини).

3NF — 2NF + відсутність транзитивних залежностей (неключовий атрибут не залежить від іншого неключового).

BCNF — посилена 3NF: кожна нетривіальна залежність є залежністю від надключа.

ACID — гарантії транзакцій B

Atomicity — атомарність: усе або нічого. Транзакція не може виконатись частково.

Consistency — узгодженість: БД переходить з одного коректного стану в інший.

Isolation — ізольованість: паралельні транзакції не впливають одна на одну.

Durability — довговічність: підтверджена транзакція зберігається навіть після збою живлення.

ER-модель та NoSQL C

ER-модель (Entity-Relationship): Сутність (Entity) — об'єкт реального світу. Атрибут — властивість. Зв'язок — відношення між сутностями. Кардинальність: 1:1, 1:M, M:N.

NoSQL: Документо-орієнтовані (MongoDB), ключ-значення (Redis), колонкові (Cassandra), графові (Neo4j).

Теорема CAP: у розподіленій системі можна забезпечити лише 2 з 3: Consistency, Availability, Partition Tolerance.

Таблиця істинності B

Вибери операцію — побачиш таблицю. A→B (імплікація) = False тільки коли A=T і B=F. Запам'ятай: «з правди не може слідувати брехня».

Симулятор ймовірностей — Закон великих чисел B

Кидай монету. При 10 кидках — результати хаотичні. При 1000 — наближаються до 50/50. Це закон великих чисел у дії.

Комбінаторика — формули B

Перестановки: P(n) = n! — скількома способами розставити n різних об'єктів у ряд.

Розміщення: A(n,k) = n!/(n−k)! — вибрати k з n, порядок важливий.

Комбінації: C(n,k) = n!/(k!(n−k)!) — вибрати k з n, порядок неважливий.

Теорія ймовірностей — ключові формули B

Умовна ймовірність: P(A|B) = P(A∩B) / P(B)

Формула повної ймовірності: P(A) = Σ P(A|Hᵢ)·P(Hᵢ)

Формула Байєса: P(H|E) = P(E|H)·P(H) / P(E) — ключова для Machine Learning!

Схема Бернуллі: P(k) = C(n,k)·pᵏ·(1−p)ⁿ⁻ᵏ — імовірність k успіхів у n випробуваннях.

Нормальний розподіл і статистика B

Нульова гіпотеза H₀ — припущення, яке перевіряємо (зазвичай «ефекту немає»). p-value — імовірність отримати такі дані, якщо H₀ вірна. p < α → H₀ відхиляється.

Лінійна алгебра (стисло) B

Множення матриць: A(m×n) × B(n×p) = C(m×p). Елемент C[i][j] = Σ A[i][k]·B[k][j].

Визначник det(A) = 0 ⇔ матриця вироджена (немає оберненої).

Власні значення λ: det(A−λI) = 0. Власний вектор v: Av = λv.

Диференціальні рівняння B

ЗДР (звичайне диф. рівняння) — містить функцію однієї змінної та її похідні.

Задача Коші — ЗДР + початкові умови.

Чисельні методи: метод Ейлера (простий, O(h)), метод Рунге-Кутти (точний, O(h⁴)), метод скінченних різниць.

Компілятор vs Інтерпретатор vs JIT B

Компілятор — перетворює весь код у машинний до запуску. Швидке виконання. C, C++, Rust, Go.

Інтерпретатор — виконує рядок за рядком під час роботи. Гнучкість. Python, Ruby, Bash.

JIT (Just-In-Time) — гібрид: компілює часто використовувані частини під час виконання. Java (JVM), C# (.NET), V8 (JS).

Парадигми програмування B

Процедурне — послідовність процедур (C). Функціональне — чисті функції, незмінні дані (Haskell). Логічне — факти + правила (Prolog). Подійно-орієнтоване — реакція на події (JS у браузері). Реактивне — автоматичне поширення змін (RxJS). Узагальнене — generics/templates.

Фундаментальні поняття B

Слабкий ШІ (Narrow AI) — спеціалізований на одній задачі (шахи, розпізнавання облич). Те, що існує зараз.

Сильний ШІ (AGI) — гіпотетичний ШІ рівня людини, здатний до будь-якої інтелектуальної задачі.

Інтелектуальний агент — система, що сприймає середовище через сенсори, діє через актуатори.

BFS/DFS у ШІ — пошук у просторі станів для розв'язання задач. Прямий/зворотний/двонаправлений пошук.

Нейромережа: класифікація «на очах» B

Клікай на полі, додаючи точки двох класів (🔵 синій та 🟠 помаранчевий). Тисни «Навчати» — мережа будує межу рішення. Фон показує впевненість: чим яскравіший — тим мережа впевненіша.

Це логістична регресія — один нейрон із сигмоїдою. Для складніших нелінійних меж потрібні додаткові шари (глибоке навчання).

Машинне навчання (ML) B

Supervised (з учителем) — дані мають мітки. Класифікація, регресія.

Unsupervised (без учителя) — міток немає. Кластеризація (k-means), PCA.

Overfitting (перенавчання) — модель «запам'ятала» тренувальні дані, погано узагальнює на нові.

Train / Validation / Test — поділ даних: ~70% / 15% / 15%.
